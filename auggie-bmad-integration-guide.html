# BMad-Method 集成规则

## 🎯 代码审查标准

### 架构合规性
- **模块化设计**: 确保代码遵循 BMad-Method 的模块化架构原则
- **依赖管理**: 检查依赖注入和模块间耦合度
- **接口设计**: 验证 API 接口设计符合 RESTful 或 GraphQL 标准
- **数据流**: 确保数据流向清晰，避免循环依赖

### 代码质量标准
- **命名规范**:
  - 类名使用 PascalCase
  - 方法名使用 camelCase
  - 常量使用 UPPER_SNAKE_CASE
  - 文件名使用 kebab-case
- **注释要求**:
  - 公共 API 必须有 JSDoc 注释
  - 复杂业务逻辑必须有行内注释
  - 每个模块必须有文件头注释
- **错误处理**:
  - 所有异步操作必须有错误处理
  - 使用统一的错误处理机制
  - 错误信息必须有意义且可追踪

### 测试要求
- **单元测试覆盖率**: 新增代码覆盖率不低于 80%
- **集成测试**: 涉及多个模块的功能必须有集成测试
- **端到端测试**: 关键用户流程必须有 E2E 测试
- **测试命名**: 测试用例名称必须清晰描述测试场景

### 安全标准
- **输入验证**: 所有用户输入必须进行验证和清理
- **权限检查**: API 端点必须有适当的权限验证
- **敏感数据**: 不得在日志中记录敏感信息
- **依赖安全**: 检查第三方依赖的安全漏洞

### 性能要求
- **响应时间**: API 响应时间不超过 200ms
- **内存使用**: 避免内存泄漏和过度内存使用
- **数据库查询**: 避免 N+1 查询问题
- **缓存策略**: 合理使用缓存提升性能

## 🔧 BMad-Method 特定规则

### 工作流合规性
- **提交信息**: 遵循 BMad-Method 提交信息格式
  ```
  type(scope): description

  [optional body]

  [optional footer]
  ```
- **分支策略**: 遵循 GitFlow 分支管理策略
- **代码审查**: 所有代码必须经过至少一人审查

### 文档要求
- **README**: 每个模块必须有完整的 README 文档
- **API 文档**: 使用 OpenAPI/Swagger 规范
- **变更日志**: 维护 CHANGELOG.md 文件
- **架构文档**: 重要架构决策必须记录在 ADR 中

### 集成检查点
- **构建状态**: 确保 CI/CD 流水线通过
- **代码质量**: SonarQube 质量门禁通过
- **安全扫描**: 安全扫描无高危漏洞
- **依赖检查**: 依赖版本符合项目要求

## ⚠️ 严重问题标记

以下问题将被标记为严重问题，必须修复：

1. **安全漏洞**: SQL 注入、XSS、CSRF 等安全问题
2. **性能问题**: 明显的性能瓶颈或资源泄漏
3. **架构违规**: 违反 BMad-Method 架构原则的设计
4. **测试缺失**: 关键功能缺少测试覆盖
5. **文档缺失**: 公共 API 缺少文档

## 📋 检查清单

### 代码提交前检查
- [ ] 代码符合项目编码规范
- [ ] 所有测试通过
- [ ] 代码覆盖率达标
- [ ] 文档已更新
- [ ] 提交信息格式正确

### Pull Request 检查
- [ ] PR 描述清晰完整
- [ ] 相关 Issue 已关联
- [ ] 代码审查已完成
- [ ] CI/CD 流水线通过
- [ ] 冲突已解决

3.2 配置选项详解

配置项 类型 默认值 说明
contextEngine boolean true 启用 Auggie 的上下文引擎，提供更准确的代码理解
autoFix boolean false 自动修复简单问题（建议在开发环境谨慎使用）
reportFormat string 'markdown' 报告格式：'markdown', 'json', 'console', 'html'
verbose boolean false 详细输出模式，显示更多调试信息
concurrency number 2 并发处理文件数量，根据系统性能调整

3.3 触发器配置最佳实践

💡 触发器策略： 不同阶段使用不同的检查强度和规则集 
// 触发器配置最佳实践
const triggerBestPractices = {
  'pre-commit': {
    // 提交前：快速检查，只检查关键问题
    enabled: true,
    rules: ['security-rules'], // 只检查安全问题
    failOnError: true,
    timeout: 15000, // 15秒快速检查
    scope: 'staged-files' // 只检查暂存的文件
  },

  'pre-push': {
    // 推送前：中等强度检查
    enabled: true,
    rules: ['bmad-integration', 'security-rules'],
    failOnError: true,
    timeout: 45000, // 45秒中等检查
    scope: 'changed-files' // 检查所有变更文件
  },

  'pr-review': {
    // PR 审查：全面检查，提供建议
    enabled: true,
    rules: ['bmad-integration', 'security-rules', 'performance-rules'],
    failOnError: false, // 不阻塞 PR，仅提供建议
    timeout: 180000, // 3分钟全面检查
    scope: 'all-files', // 检查所有相关文件
    generateReport: true // 生成详细报告
  }
};

CI/CD 集成

配置 GitHub Actions 工作流

4.1 创建主要工作流文件

创建 .github/workflows/bmad-auggie-integration.yml：

# BMad-Method 与 Auggie CLI 集成工作流
name: 🔄 BMad-Auggie Integration

# 触发条件
on:
  # Pull Request 触发
  pull_request:
    branches:
      - main
      - develop
    paths-ignore:
      - '**.md'
      - 'docs/**'
      - '.gitignore'

  # Push 到主分支触发
  push:
    branches:
      - main
      - develop

  # 手动触发
  workflow_dispatch:
    inputs:
      environment:
        description: '运行环境'
        required: true
        default: 'development'
        type: choice
        options:
          - development
          - staging
          - production

# 环境变量
env:
  NODE_VERSION: '22'
  AUGGIE_VERSION: 'latest'
  BMAD_VERSION: 'latest'

# 作业定义
jobs:
  # 环境准备和依赖安装
  setup:
    name: 🚀 环境准备
    runs-on: ubuntu-latest
    outputs:
      node-version: ${{ steps.setup-node.outputs.node-version }}
      cache-key: ${{ steps.cache-deps.outputs.cache-hit }}

    steps:
      - name: 📥 检出代码
        uses: actions/checkout@v4
        with:
          # 获取完整历史，用于 BMad-Method 分析
          fetch-depth: 0

      - name: 🟢 设置 Node.js
        id: setup-node
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: 📦 缓存依赖
        id: cache-deps
        uses: actions/cache@v3
        with:
          path: |
            ~/.npm
            node_modules
            .augment/cache
          key: ${{ runner.os }}-deps-${{ hashFiles('**/package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-deps-

      - name: 🔧 安装全局依赖
        run: |
          npm install -g @augmentcode/auggie@${{ env.AUGGIE_VERSION }}
          npx bmad-method@${{ env.BMAD_VERSION }} install

      - name: ✅ 验证安装
        run: |
          echo "Node.js 版本: $(node --version)"
          echo "npm 版本: $(npm --version)"
          echo "Auggie CLI 版本: $(auggie --version)"
          echo "BMad-Method 版本: $(bmad-method --version)"

  # 代码质量检查
  code-quality:
    name: 🔍 代码质量检查
    runs-on: ubuntu-latest
    needs: setup

    steps:
      - name: 📥 检出代码
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: 🟢 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: 📦 恢复依赖缓存
        uses: actions/cache@v3
        with:
          path: |
            ~/.npm
            node_modules
            .augment/cache
          key: ${{ runner.os }}-deps-${{ hashFiles('**/package-lock.json') }}

      - name: 🔧 安装依赖
        run: |
          npm ci
          npm install -g @augmentcode/auggie@${{ env.AUGGIE_VERSION }}
          npx bmad-method@${{ env.BMAD_VERSION }} install

      - name: 🧹 代码格式检查
        run: |
          npm run lint
          npm run format:check

      - name: 🏗️ 构建检查
        run: |
          npm run build

      - name: 🧪 运行测试
        run: |
          npm run test:coverage

      - name: 📊 上传覆盖率报告
        uses: codecov/codecov-action@v3
        with:
          file: ./coverage/lcov.info
          flags: unittests
          name: codecov-umbrella

  # BMad-Method 验证
  bmad-validation:
    name: 🧠 BMad-Method 验证
    runs-on: ubuntu-latest
    needs: setup

    steps:
      - name: 📥 检出代码
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: 🟢 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: 🔧 安装 BMad-Method
        run: |
          npx bmad-method@${{ env.BMAD_VERSION }} install

      - name: ✅ BMad 项目验证
        run: |
          bmad-method validate --strict

      - name: 📚 更新知识库
        run: |
          bmad-method update-kb --auto-commit=false

      - name: 📋 生成 BMad 报告
        run: |
          bmad-method report --format=json --output=bmad-report.json

      - name: 📤 上传 BMad 报告
        uses: actions/upload-artifact@v3
        with:
          name: bmad-report
          path: bmad-report.json

  # Auggie CLI 代码审查
  auggie-review:
    name: 🤖 Auggie CLI 审查
    runs-on: ubuntu-latest
    needs: setup

    steps:
      - name: 📥 检出代码
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: 🟢 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: 🔧 安装 Auggie CLI
        run: |
          npm install -g @augmentcode/auggie@${{ env.AUGGIE_VERSION }}

      - name: 🔍 安全规则检查
        run: |
          auggie --rules .augment/rules/security-rules.md \
                 --format json \
                 --output security-report.json \
                 --fail-on-error

      - name: ⚡ 性能规则检查
        run: |
          auggie --rules .augment/rules/performance-rules.md \
                 --format json \
                 --output performance-report.json \
                 --no-fail-on-error

      - name: 🏗️ BMad 集成规则检查
        run: |
          auggie --rules .augment/rules/bmad-integration.md \
                 --format json \
                 --output integration-report.json \
                 --context-engine \
                 --fail-on-error

      - name: 📤 上传 Auggie 报告
        uses: actions/upload-artifact@v3
        with:
          name: auggie-reports
          path: |
            security-report.json
            performance-report.json
            integration-report.json

  # 集成审查
  integrated-review:
    name: 🔗 集成审查
    runs-on: ubuntu-latest
    needs: [bmad-validation, auggie-review]

    steps:
      - name: 📥 检出代码
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: 🟢 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: 🔧 安装工具
        run: |
          npm install -g @augmentcode/auggie@${{ env.AUGGIE_VERSION }}
          npx bmad-method@${{ env.BMAD_VERSION }} install

      - name: 📥 下载报告
        uses: actions/download-artifact@v3
        with:
          name: bmad-report
          path: ./reports/

      - name: 📥 下载 Auggie 报告
        uses: actions/download-artifact@v3
        with:
          name: auggie-reports
          path: ./reports/

      - name: 🔄 运行集成工作流
        run: |
          bmad-method workflow run codeReview \
            --config bmad-config.js \
            --environment ${{ github.event.inputs.environment || 'development' }}

      - name: 📊 生成综合报告
        run: |
          node scripts/generate-integrated-report.js \
            --bmad-report ./reports/bmad-report.json \
            --auggie-reports ./reports/ \
            --output ./reports/integrated-report.md

      - name: 💬 PR 评论
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v6
        with:
          script: |
            const fs = require('fs');
            const report = fs.readFileSync('./reports/integrated-report.md', 'utf8');

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## 🔍 集成代码审查报告\n\n${report}`
            });

      - name: 📤 上传综合报告
        uses: actions/upload-artifact@v3
        with:
          name: integrated-report
          path: ./reports/integrated-report.md

4.2 创建报告生成脚本

创建 scripts/generate-integrated-report.js：

#!/usr/bin/env node

const fs = require('fs');
const path = require('path');
const { program } = require('commander');

// 命令行参数解析
program
  .option('--bmad-report ', 'BMad-Method 报告路径')
  .option('--auggie-reports ', 'Auggie CLI 报告目录')
  .option('--output ', '输出文件路径')
  .parse();

const options = program.opts();

// 生成集成报告
async function generateIntegratedReport() {
  try {
    // 读取 BMad-Method 报告
    const bmadReport = JSON.parse(
      fs.readFileSync(options.bmadReport, 'utf8')
    );

    // 读取 Auggie CLI 报告
    const auggieReports = {
      security: JSON.parse(
        fs.readFileSync(path.join(options.auggieReports, 'security-report.json'), 'utf8')
      ),
      performance: JSON.parse(
        fs.readFileSync(path.join(options.auggieReports, 'performance-report.json'), 'utf8')
      ),
      integration: JSON.parse(
        fs.readFileSync(path.join(options.auggieReports, 'integration-report.json'), 'utf8')
      )
    };

    // 生成 Markdown 报告
    const report = generateMarkdownReport(bmadReport, auggieReports);

    // 写入输出文件
    fs.writeFileSync(options.output, report, 'utf8');

    console.log(`✅ 集成报告已生成: ${options.output}`);
  } catch (error) {
    console.error('❌ 生成报告失败:', error.message);
    process.exit(1);
  }
}

function generateMarkdownReport(bmadReport, auggieReports) {
  const timestamp = new Date().toISOString();

  return `# 🔍 集成代码审查报告

**生成时间**: ${timestamp}

## 📊 总体概览

| 工具 | 状态 | 问题数量 | 严重程度 |
|------|------|----------|----------|
| BMad-Method | ${bmadReport.status} | ${bmadReport.issues?.length || 0} | ${bmadReport.severity || 'N/A'} |
| Auggie (安全) | ${auggieReports.security.status} | ${auggieReports.security.issues?.length || 0} | ${auggieReports.security.severity || 'N/A'} |
| Auggie (性能) | ${auggieReports.performance.status} | ${auggieReports.performance.issues?.length || 0} | ${auggieReports.performance.severity || 'N/A'} |
| Auggie (集成) | ${auggieReports.integration.status} | ${auggieReports.integration.issues?.length || 0} | ${auggieReports.integration.severity || 'N/A'} |

## 🧠 BMad-Method 分析结果

${formatBmadReport(bmadReport)}

## 🤖 Auggie CLI 分析结果

### 🔒 安全检查
${formatAuggieReport(auggieReports.security)}

### ⚡ 性能检查
${formatAuggieReport(auggieReports.performance)}

### 🔗 集成检查
${formatAuggieReport(auggieReports.integration)}

## 💡 综合建议

${generateRecommendations(bmadReport, auggieReports)}

---
*此报告由 BMad-Method 与 Auggie CLI 集成系统自动生成*`;
}

function formatBmadReport(report) {
  if (!report.issues || report.issues.length === 0) {
    return '✅ 未发现问题';
  }

  return report.issues.map(issue =>
    `- **${issue.severity}**: ${issue.message} (${issue.file}:${issue.line})`
  ).join('\n');
}

function formatAuggieReport(report) {
  if (!report.issues || report.issues.length === 0) {
    return '✅ 未发现问题';
  }

  return report.issues.map(issue =>
    `- **${issue.severity}**: ${issue.message}\n  - 文件: ${issue.file}\n  - 建议: ${issue.suggestion || '无'}`
  ).join('\n\n');
}

function generateRecommendations(bmadReport, auggieReports) {
  const recommendations = [];

  // 基于报告生成建议
  const totalIssues = (bmadReport.issues?.length || 0) +
                     (auggieReports.security.issues?.length || 0) +
                     (auggieReports.performance.issues?.length || 0) +
                     (auggieReports.integration.issues?.length || 0);

  if (totalIssues === 0) {
    recommendations.push('🎉 代码质量良好，符合 BMad-Method 标准！');
  } else {
    recommendations.push(`📋 发现 ${totalIssues} 个问题需要处理`);

    if (auggieReports.security.issues?.length > 0) {
      recommendations.push('🔒 优先处理安全问题');
    }

    if (bmadReport.issues?.length > 0) {
      recommendations.push('🧠 确保代码符合 BMad-Method 架构原则');
    }

    if (auggieReports.performance.issues?.length > 0) {
      recommendations.push('⚡ 考虑优化性能相关问题');
    }
  }

  return recommendations.join('\n- ');
}

// 运行脚本
generateIntegratedReport();
